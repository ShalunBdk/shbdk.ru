# Восстановление системного раздела Windows

> [!warning]
> Перед выполнением любых операций с дисками и разделами рекомендуется создать резервную копию важных данных. Неправильные действия могут привести к потере информации.

## Обзор процедуры

Если загрузочный раздел Windows поврежден или отсутствует, вы можете пересоздать его и восстановить загрузочные файлы с использованием утилиты **bcdboot**. Данная инструкция описывает подробный процесс восстановления работоспособности системы.

## Предварительные требования

<div class="custom-container tip">
  <p class="custom-container-title">Необходимые инструменты</p>
  <ul>
    <li>Установочный носитель Windows (USB-накопитель или DVD-диск)</li>
    <li>Доступ к командной строке через среду восстановления Windows (WinRE)</li>
  </ul>
</div>

## Пошаговая инструкция

### 1. Загрузка в среду восстановления Windows

> [!note]
>
> 1. Вставьте установочный носитель Windows и перезагрузите компьютер
> 2. При появлении соответствующего сообщения нажмите любую клавишу для загрузки с установочного носителя
> 3. Выберите предпочтительный язык, раскладку клавиатуры и нажмите **Далее**
> 4. Нажмите **Восстановление системы** в левом нижнем углу экрана
> 5. Выберите **Устранение неполадок** → **Командная строка**

### 2. Работа с разделами диска через DiskPart

::: warning Будьте внимательны!
Команды DiskPart могут привести к потере данных при неправильном использовании. Внимательно проверяйте номера дисков и разделов перед выполнением команд удаления или форматирования.
:::

1. Запустите утилиту управления дисками:

```cmd
diskpart
```

2. Отобразите список доступных дисков:

```cmd
list disk
```

3. Выберите диск, на котором находится система Windows:

```cmd
select disk 0
```

::: tip Примечание
Замените `0` на номер вашего диска, если он отличается
:::

4. Отобразите разделы выбранного диска:

```cmd
list partition
```

5. Удаление поврежденного загрузочного раздела:

```cmd
select partition 1
delete partition override
```

::: tip Примечание
Замените `1` на номер вашего загрузочного раздела
:::

6. Создание нового загрузочного раздела:

::: tabs
@tab Legacy BIOS

```cmd
create partition primary size=500
format fs=ntfs quick
active
assign letter=S
```

@tab UEFI

```cmd
create partition primary size=500
format fs=fat32 quick
active
assign letter=S
```

:::

7. Выйдите из DiskPart:

```cmd
exit
```

### 3. Восстановление загрузочных файлов Windows

1. Проверка наличия системных файлов:

```cmd
dir C:\Windows
```

::: tip Если диск не C:
Если системный диск имеет другую букву, назначьте ему букву C:

```cmd
diskpart
list vol
select vol 2
assign letter=C
exit
```

Замените `2` на номер вашего системного тома
:::

2. Восстановление загрузочных файлов:

::: tabs
@tab Legacy BIOS

```cmd
bcdboot C:\Windows /s S: /f BIOS
```

@tab UEFI

```cmd
bcdboot C:\Windows /s S: /f UEFI
```

:::

::: info Параметры команды bcdboot

- **C:\\Windows** — путь к установленной Windows
- **/s S:** — буква нового загрузочного раздела
- **/f BIOS/UEFI** — указание типа загрузки (BIOS или UEFI)
  :::

### 4. Перезагрузка и проверка работоспособности

1. Перезагрузите компьютер:

```cmd
exit
```

2. Извлеките установочный носитель и убедитесь, что система загружается корректно

## Особенности и рекомендации

::: tip UEFI vs Legacy BIOS

- Для систем с **UEFI** загрузочный раздел должен быть отформатирован в **FAT32**
- Для систем с **Legacy BIOS** загрузочный раздел должен быть отформатирован в **NTFS**
  :::

::: details Устранение проблем
Если восстановление завершилось с ошибкой:

1. Выполните проверку диска:

   ```cmd
   chkdsk S: /f
   ```

2. Убедитесь, что выбран правильный путь к Windows:

   ```cmd
   dir C:\Windows\System32
   ```

3. Проверьте, что загрузочный раздел правильно отформатирован (FAT32 для UEFI, NTFS для BIOS)
   :::

## Диагностика загрузки

::: details Проблемы с загрузкой после восстановления
Если загрузка по-прежнему невозможна, проверьте настройки BIOS/UEFI:

1. Войдите в BIOS/UEFI (обычно клавиши F2, Del, F10 или Esc при запуске)
2. Убедитесь, что выбран правильный загрузочный диск
3. Проверьте режим загрузки (UEFI или Legacy), он должен соответствовать типу созданного раздела
4. Проверьте порядок загрузки (Boot Order)
5. Сохраните изменения и перезагрузите компьютер
   :::

::: details Восстановление загрузчика без пересоздания раздела
Если загрузочный раздел не поврежден, но загрузчик не работает, можно восстановить его без создания нового раздела:

1. Загрузитесь с установочного носителя Windows
2. Откройте командную строку
3. Найдите букву системного раздела (обычно C:) и загрузочного раздела
4. Выполните команду:
   ```cmd
   bcdboot C:\Windows /s S: /f ALL
   ```
   где S: — буква загрузочного раздела
   :::

::: details Дополнительные команды для восстановления
Иногда требуется восстановление загрузчика с использованием дополнительных команд:

1. Восстановление загрузчика (MBR) для Legacy BIOS:

   ```cmd
   bootrec /fixmbr
   bootrec /fixboot
   bootrec /rebuildbcd
   ```

2. Восстановление загрузчика для UEFI:
   ```cmd
   bcdboot C:\Windows /s S: /f UEFI /addlast
   ```
   :::
