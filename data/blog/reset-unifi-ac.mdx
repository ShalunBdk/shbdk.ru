# Сброс и сервисный режим Unifi точек доступа

## Интерпретация световой индикации

Перед выполнением любых действий полезно понимать, что означают различные режимы мигания светодиодных индикаторов на устройствах Unifi.

::: tip Справочная информация
Подробное руководство по интерпретации световой индикации доступно в [официальной документации Ubiquiti](https://help.ui.com/hc/en-us/articles/204910134-Understanding-Device-LED-Status-Indicators).
:::

## Методы сброса точек доступа Ubiquiti

Существует два основных способа сброса точек доступа Unifi к заводским настройкам:

### 1. Удаленный сброс через SSH

::: warning Необходимые условия
Для использования данного метода требуется:

- Активная сеть и подключение к точке доступа
- Логин и пароль для SSH-доступа
- SSH-клиент на вашем компьютере
  :::

::: details Пошаговая инструкция сброса через SSH

1. Запустите SSH-клиент (например, PuTTY, Terminal, PowerShell)
2. Подключитесь к IP-адресу точки доступа
3. Введите учетные данные контроллера Unifi для авторизации
4. После успешного входа выполните команду:
   ```bash
   syswrapper.sh restore-default
   ```
5. Дождитесь перезагрузки устройства
   :::

### 2. Физический сброс с помощью кнопки Reset

::: info Расположение кнопки
В зависимости от модели точки доступа, кнопка Reset может располагаться на задней или нижней части устройства и обычно утоплена в корпус для предотвращения случайного нажатия.
:::

<div class="custom-container tip">
  <p class="custom-container-title">Необходимые инструменты</p>
  <p>
    Для нажатия утопленной кнопки потребуется тонкий предмет, например распрямленная канцелярская
    скрепка или штифт для извлечения SIM-карты.
  </p>
</div>

::: details Пошаговая инструкция физического сброса

1. Подготовьте тонкий инструмент для нажатия кнопки Reset
2. Найдите утопленную кнопку Reset на корпусе устройства
3. Нажмите и удерживайте кнопку в течение **10 секунд**
4. Отпустите кнопку — световые индикаторы должны погаснуть
5. Дождитесь перезагрузки устройства (1-2 минуты)
6. После успешного сброса световой индикатор должен загореться оранжевым цветом
   :::

## Сервисный режим для обновления прошивки

В некоторых случаях может потребоваться ручное обновление прошивки точки доступа через сервисный режим (Recovery Mode), особенно при проблемах с обычным обновлением через контроллер.

::: tip Официальная документация
Подробное описание процесса доступно в [официальной документации Ubiquiti по Recovery Mode](https://help.ui.com/hc/en-us/articles/360043360253-UniFi-Recovery-Mode).
:::

### Подготовка к обновлению

1. Скачайте необходимую версию прошивки с [официального сайта Ubiquiti](https://ui.com/download/releases/firmware)

2. Обеспечьте сетевое соединение между вашим компьютером и точкой доступа одним из способов:
   - Подключение через коммутатор PoE
   - Прямое подключение к порту данных на адаптере PoE
   - Подключение к тому же сегменту сети/VLAN

### Вход в сервисный режим

::: warning Внимание!
Соблюдайте точную последовательность действий. Прерывание процесса может привести к повреждению прошивки устройства.
:::

::: details Пошаговая инструкция входа в сервисный режим

1. Нажмите и удерживайте кнопку Reset на устройстве
2. Не отпуская кнопку, подключите устройство к источнику питания (PoE)
3. Продолжайте удерживать кнопку до тех пор, пока светодиод не начнет мигать белым/синим или не погаснет (согласно [руководству по индикации](https://help.ui.com/hc/en-us/articles/204910134#:~:text=Flashing%20White%2DBlue%2DOff))
4. Отпустите кнопку — устройство теперь находится в режиме восстановления TFTP
   :::

### Настройка сетевого подключения

::: info IP-адрес сервисного режима
В сервисном режиме устройство использует статический IP-адрес **192.168.1.20**
:::

Установите на вашем компьютере статический IP-адрес для связи с устройством:

```
Static IP: 192.168.1.25
Subnet: 255.255.255.0
Gateway: 192.168.1.20
```

### Передача прошивки по TFTP

::: tabs
@tab Windows

1. Скачайте и установите программу Tftpd64
2. Настройте программу:
   - В поле "Server interface" выберите IP-адрес вашего компьютера (192.168.1.25)
   - В поле "Current directory" укажите путь к папке с файлом прошивки
3. Выберите загруженный файл прошивки
4. Нажмите кнопку "Put" для отправки файла на устройство

@tab macOS/Linux

1. Откройте Терминал
2. Запустите TFTP-клиент командой:
   ```bash
   tftp
   ```
3. Введите последовательно следующие команды:
   ```bash
   connect 192.168.1.20
   binary
   rexmt 1
   timeout 60
   put /path/to/firmware_name.bin
   ```
   > Замените `/path/to/firmware_name.bin` фактическим путем к файлу прошивки
   > :::

### Завершение процесса обновления

::: info Важно!
После начала процесса обновления:

- Не отключайте питание устройства
- Не прерывайте сетевое соединение
- Не перезагружайте устройство вручную
  :::

1. После успешной передачи файла начнется процесс обновления прошивки
2. Дождитесь автоматической перезагрузки устройства
3. После перезагрузки точка доступа должна загрузиться с новой прошивкой

## Устранение проблем

::: details Устройство не входит в сервисный режим

- Убедитесь, что вы удерживаете кнопку Reset достаточно долго (не менее 10 секунд)
- Проверьте, что устройство получает питание
- Попробуйте использовать другой инструмент для нажатия кнопки Reset
- Некоторые модели имеют другую последовательность входа в сервисный режим, проверьте документацию для конкретной модели
  :::

::: details Ошибка при передаче файла по TFTP

- Проверьте сетевое подключение между компьютером и устройством
- Убедитесь, что статический IP-адрес настроен правильно
- Проверьте, что файл прошивки соответствует вашей модели устройства
- Попробуйте уменьшить размер MTU на сетевом интерфейсе компьютера до 1400
  :::

::: details Устройство не перезагружается после обновления

- Подождите не менее 5 минут
- Если устройство не перезагружается, выполните принудительную перезагрузку, отключив и снова подключив питание
- Проверьте световую индикацию для диагностики состояния
  :::
